## Why

当前的需求输入页面（IntakeView）提供两种模式："群消息录入"（粘贴大段文本到 textarea）和"结构化需求表"（手动填写数据表格）。这两种方式都是批量式的——用户需要先在外部准备好数据，粘贴/录入后点击"生成推荐"，整个流程缺乏与项目经理智能体的互动感。用户希望将其重构为对话式交互：用户可以像与项目经理聊天一样逐步输入需求、追问、补充细节，由智能体实时响应并结构化需求。这能显著降低使用门槛，让需求录入过程更自然、更灵活。

## What Changes

- 将 IntakeView 的需求输入区域从 textarea/数据表格重构为聊天对话界面
- 用户以消息形式逐条发送需求描述，智能体实时解析并反馈结构化结果
- 保留"生成推荐"作为对话完成后的总结性操作
- 保留人员说明、模块知识库导入和解析结果预览面板不变
- 后端新增流式/多轮对话 API，支持逐条消息发送与增量解析结果返回

## Capabilities

### New Capabilities
- `chat-style-intake`: 对话式需求输入交互，支持用户逐条发送需求、智能体实时解析反馈、对话历史维护

## Impact

- **前端**: `IntakeView.vue` 需要大幅重写，从当前双模式表单改为聊天界面组件
- **后端**: 需要新增对话式需求解析 API（当前 `saveDraft` 和 `generateRecommendations` 是批量操作）
- **状态管理**: `workspace.js` 需要新增对话消息状态和相关 API 调用
- **API 客户端**: 需要新增对话相关接口
- 当前 `draft` 的 `message_text` 和 `requirement_rows` 字段可能需要重新设计或保留兼容
