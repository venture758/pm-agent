## Why

当前项目经理 Agent 只提供 Python 接口和离线调用方式，实际使用时需要手工组织输入、执行脚本并自行比对输出，不适合项目经理、组长或测试负责人以页面方式完成日常需求分配和风险跟踪。现在需要补充一个前后端分离的 Web 工作台，前端使用 Vue 承接交互，后端暴露清晰 API 复用现有 Agent 能力，让需求录入、知识库导入、分配确认、Excel 同步和执行监控都能在浏览器中完成，降低使用门槛并减少人工串联步骤。

## What Changes

- 新增一个面向项目经理 Agent 的前后端分离 Web 工作台，前端使用 Vue，承接需求录入、成员维护、模块知识库导入、分配推荐、确认调整、Excel 同步、执行预警和团队洞察。
- 新增会话式工作区，将一次页面操作中的需求、成员、知识库、推荐结果、确认结果和同步批次保存在同一工作上下文中，避免每一步都重新组织输入。
- 新增前后端交互契约，使 Vue 前端通过 API 调用现有 Agent 能力，并返回结构化结果、校验错误和操作状态。
- 新增文件上传与结果展示交互，支持通过页面上传模块知识库 Excel、故事 Excel 和任务 Excel，并在页面内查看同步结果、预警和负载摘要。
- 新增推荐确认页面交互，支持项目经理在页面上采纳、改派、拆分需求、添加协作人，并查看推荐依据与变更后的平台落地结果。

## Capabilities

### New Capabilities
- `project-manager-agent-web-workbench`: 提供基于 Vue 前端和后端 API 的项目经理工作台，覆盖需求输入、知识库导入、分配推荐、确认调整、执行监控和团队洞察的完整交互流程。

### Modified Capabilities
- None.

## Impact

- Affected code: 需要新增后端 API 应用入口、请求校验层、工作区状态管理层、Vue 前端工程、页面路由、状态管理、上传组件，以及对现有 `pm_agent` 工作流的适配调用。
- APIs: 需要定义前端调用的后端接口，覆盖需求录入、成员编辑、知识库上传、推荐生成、确认提交、平台落地预览、Excel 同步、预警查询和团队洞察查询。
- Dependencies: 可能需要引入 Python Web API 框架、文件上传与表单校验依赖，以及 Vue、路由和 HTTP 客户端相关依赖。
- Systems: 影响当前仅脚本式调用的使用方式，新增前端单页应用和后端 API 服务，并要求后端在单次工作会话中维护 Agent 状态和文件导入结果。
