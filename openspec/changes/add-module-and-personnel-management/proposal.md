## Why

当前工作台对业务模块和成员信息的处理仍以 Excel 导入或需求处理时的临时填写为主，缺少稳定的维护入口，导致模块负责人、B 角、熟悉度和人员画像难以持续更新。现在需要新增业务模块维护和人员管理维护功能，并让人员维护数据成为唯一人员来源，减少对离线文件和 intake 临时成员输入的依赖，提升推荐结果的可维护性和准确性。

## What Changes

- 新增业务模块维护能力，支持在工作台中查看、新增、编辑、删除业务模块条目，并维护大模块、功能模块、负责人、B 角和成员熟悉度等信息。
- 新增人员管理维护能力，支持在工作台中查看、新增、编辑、删除成员信息，并维护角色、技能、经验、负载、容量和约束等画像字段。
- 新增前后端维护接口和工作区状态回写机制，使模块与成员维护结果可以被推荐、监控和洞察流程直接复用。
- 去掉 intake 阶段的临时成员来源，推荐等流程统一从人员管理维护数据读取团队成员信息。
- 保留现有 Excel 导入能力，但将其定位为批量导入入口，与页面维护能力并存而不是唯一更新方式。
- 补充维护操作的校验、冲突提示和测试，避免重复模块键、空成员名或无效容量等数据破坏后续流程。

## Capabilities

### New Capabilities
- `business-module-maintenance`: 提供业务模块知识条目的页面维护与 API 维护能力，覆盖模块新增、编辑、删除、查询和校验。
- `personnel-management`: 提供团队成员画像的页面维护与 API 维护能力，覆盖成员新增、编辑、删除、查询和校验。

### Modified Capabilities
- None.

## Impact

- Affected code: 需要扩展 `pm_agent` 的工作区服务、状态模型和 API 路由，并更新 Vue 前端路由、导航、状态管理和维护页面组件。
- APIs: 需要新增业务模块维护和人员管理维护相关的 REST 接口，支持列表读取与保存操作。
- Dependencies: 无强制新增依赖，优先复用现有 Python 后端与 Vue 前端栈。
- Systems: 影响模块知识库和成员画像的维护方式，但不改变现有推荐、确认、同步、监控与洞察的核心业务目标。
