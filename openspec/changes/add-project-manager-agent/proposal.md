## 背景与目标

软件研发团队在做需求排期和任务分配时，往往需要项目经理同时理解需求复杂度、优先级、风险、成员技能、当前负载和可用时间，这个过程耗时且容易受主观判断影响。现在需要一个项目经理 Agent，在接收到需求列表和团队成员情况后，自动生成合理的需求分配方案，并在执行过程中持续发现风险和资源问题，降低人工协调成本并提升交付稳定性。

## 变更内容

- 新增一个面向软件研发团队的项目经理 Agent，能够接收产品经理在群里发送的待评审需求列表，也能够通过导入方式读取需求数据。
- 新增需求分析能力，自动识别需求类型、复杂度、风险、依赖和拆分机会。
- 新增团队成员建模能力，整理成员角色、技能标签、经验、当前负载、可用容量和特殊约束。
- 新增自动分配推荐能力，为每条需求生成负责人、备选人、测试负责人和拆分建议，并产出贴近群回复习惯的推荐文本。
- 新增项目经理确认与平台落地能力，支持采纳推荐、改派、拆分需求、指定协作人，并将确认结果映射到项目管理平台的故事和任务数据。
- 新增执行监控能力，基于每天人工导出的故事列表和任务列表 Excel 识别延期、阻塞和质量异常，并触发预警和调整建议。
- 新增离线同步能力，在缺少项目平台查询接口的前提下，按 `用户故事编码` 和 `任务编号` 对导入数据执行新增或更新。
- 新增团队负载与能力管理能力，输出成员负载热力图、单点依赖识别和新人培养建议。
- 新增业务模块知识库能力，基于“模块划分、主要负责人、B角、成员熟悉程度”Excel 建立可维护的模块知识库，并在每次需求分配后自动更新。
- 新增 Agent 配置与输出约束，使结果保持可执行、可复核，并便于后续人工调整。

## 能力范围

### 新增能力
- `requirement-intake-and-analysis`: 将群聊或导入输入的需求列表和团队成员信息整理为可分配的结构化输入，并识别需求类型、复杂度、风险和拆分建议。
- `assignment-recommendation-and-platform-handoff`: 基于需求特征、技能匹配和成员负载生成分配方案，支持项目经理采纳、改派、拆分和指定协作人，并输出故事/任务落地结果。
- `execution-monitoring-and-risk-adjustment`: 根据项目管理平台中的故事、任务状态及执行信号识别异常，触发预警并给出调整建议。
- `team-load-and-capability-management`: 输出团队负载热力图、单点依赖识别和能力培养建议，辅助长期资源管理。
- `module-knowledge-base-management`: 建立并维护业务模块知识库，管理模块负责人、B角、成员熟悉度和分配历史。

### 变更能力

- None.

## 影响范围

- Affected code: Agent 核心编排、群消息解析模块、需求/成员/故事/任务/执行状态数据模型、模块知识库模型、分配策略模块、监控预警模块、输出格式化模块、测试用例。
- APIs: 需要定义输入输出契约，覆盖需求导入、群消息解析、成员信息输入、模块知识库导入、分配确认、故事/任务 Excel 导入、基于编码的新增或更新、执行状态更新和负载洞察输出。
- Dependencies: 可能需要引入结构化输出校验、规则评分和容量分析支持。
- Systems: 影响命令行或服务层调用入口，以及后续与群消息来源、Excel 导入流程、人力台账、工单系统、质量平台和内部知识库存储的集成接口。
